# Library Report Command

Generate a comprehensive report of the RIZQ dua library status in Firebase Firestore.

## Report Sections

### 1. Executive Summary
- Total duas in library
- Total journeys available
- Content coverage percentage
- Quality score

### 2. Collection Statistics
- Duas count and breakdown
- Journeys count and status
- Categories utilization
- Journey-dua mappings

### 3. Content Analysis
- Category distribution
- Difficulty distribution
- XP distribution
- Time slot coverage

### 4. Quality Metrics
- Completeness score
- Fields populated percentage
- Authentication rate (sources verified)
- Prophetic context coverage

### 5. Gap Analysis
- Missing content areas
- Underrepresented categories
- Journey improvement suggestions

## Data Collection

Query Firebase for:
```
1. All duas - get count, aggregate stats
2. All journeys - get count, featured status
3. All journey_duas - get mappings
4. All categories - verify all exist
```

## Health Score Calculation

```javascript
function calculateHealthScore(data) {
  let score = 0;

  // Has minimum duas (25 points)
  if (data.duas.length >= 10) score += 25;
  else score += (data.duas.length / 10) * 25;

  // Category balance (25 points)
  const categoryBalance = calculateCategoryBalance(data.duas);
  score += categoryBalance * 25;

  // Field completeness (25 points)
  const completeness = calculateCompleteness(data.duas);
  score += completeness * 25;

  // Journey coverage (25 points)
  if (data.journeys.length >= 3) score += 25;
  else score += (data.journeys.length / 3) * 25;

  return Math.round(score);
}
```

## Output Options

Ask user preferred output:
1. **Console** - Display in terminal
2. **Markdown file** - Save to `RIZQ-iOS/docs/library-report.md`
3. **Both** - Display and save

## Scheduling

Suggest generating reports:
- After batch additions
- Before app releases
- Weekly during active development
